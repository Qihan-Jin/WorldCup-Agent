# MatchPilot 产品设计规格中文版

## 产品概述

MatchPilot 是一个面向普通球迷的移动端优先 AI 世界杯观赛规划 Agent。它帮助时间有限、但想参与 2026 世界杯的用户，根据主队、可用时间和看球偏好，筛选值得观看的比赛，并生成个性化观赛计划。

产品标语：

> Set your team. Plan your World Cup.

中文理解：

> 选定你的主队，规划你的世界杯。

MatchPilot 不只是赛程表，也不是泛聊天机器人。它更像一个“球迷观赛规划助手”：先理解用户是谁、支持谁、什么时候方便看球，再帮用户判断哪些比赛必看、哪些适合看集锦、哪些可以跳过，并提供赛前看点、比赛提醒和 AI 问答。

## 目标用户

核心用户是普通球迷和轻度球迷：

- 想看世界杯，但无法每场都追。
- 有喜欢的球队、球星或地区。
- 不太清楚哪些比赛最值得看。
- 想在赛前快速理解比赛看点。
- 想收到重要比赛的开赛提醒。
- 想和朋友聊球，但不想花太多时间做功课。

重度球迷也可以使用 MatchPilot，但第一版不以专业战术分析为核心，而是优先服务“时间有限但想参与世界杯氛围”的普通用户。

## 核心问题

世界杯期间信息很多，但对普通球迷来说常常是碎片化的。用户真正想解决的是：

- 我今晚应该看哪场？
- 我的主队下一场什么时候踢？
- 哪些比赛值得熬夜？
- 哪些比赛可以只看集锦？
- 这场比赛为什么重要？
- 比赛快开始时能不能提醒我？

MatchPilot 的价值是把原始赛程和用户偏好转化为清晰、可执行的个人观赛计划。

## MVP 范围

第一版做成移动端优先的网页应用或 PWA，体验上接近小程序。

第一版包含：

- 主队设置。
- 用户偏好设置，包括可看球时间、熬夜接受度、看球深度、提醒偏好。
- 首页展示主队下一场、推荐比赛、提醒状态。
- 个性化观赛计划，分为必看、推荐、看集锦、可跳过。
- 比赛详情页，展示重要性评分、推荐理由、赛前看点和提醒设置。
- Ask AI 页面，用于调整计划和回答观赛相关问题。
- 基于浏览器的本地提醒模拟，使用 localStorage、定时检查和可选 Web Notification。
- 默认使用 mock 世界杯赛程数据，保证 Demo 稳定运行。
- 预留足球数据 API 接入层，后续可切换到真实数据源。
- 配套产品文档、Agent 工作流文档和 API 设计文档。

第一版不做：

- 真实账号登录。
- 服务端推送通知。
- 支付功能。
- 完整社区功能。
- 分钟级实时文字直播。
- 博彩、赔率或赌博相关功能。
- 付费级战术数据或高级球员指标。
- 正式发布微信小程序。

## 用户路径

1. 用户在手机端打开 MatchPilot。
2. 用户选择自己的主队，也可以选择“我还没有主队，帮我推荐”。
3. 用户设置看球偏好，比如是否能熬夜、哪些时间方便、自己看球深度如何。
4. MatchPilot 生成个性化世界杯首页。
5. 用户看到主队下一场比赛和近期最值得关注的比赛。
6. 用户进入观赛计划页，查看按优先级分组的比赛。
7. 用户进入比赛详情页，了解比赛为什么重要，并设置提醒。
8. 用户可以在 Ask AI 页面提出调整要求，例如“只看周末比赛”“我这周不能熬夜”“只关注阿根廷”。

## 页面设计

### Onboarding 设置页

目标：收集足够信息，用于个性化推荐。

关键内容：

- 主队选择。
- “我还没有主队”选项。
- 可看球时间。
- 熬夜接受度。
- 看球深度。
- 是否喜欢和朋友一起看球。
- 默认提醒偏好。

### Home 首页

目标：成为用户每天打开的世界杯入口。

关键内容：

- 主队下一场比赛。
- 今日或近期推荐比赛。
- 比赛卡片，包含重要性评分、开赛时间、推荐理由。
- 已设置提醒。
- 进入 AI 观赛计划的入口。

### Plan 观赛计划页

目标：展示 AI 生成的个人世界杯观赛计划。

关键内容：

- 计划摘要。
- 必看比赛。
- 推荐比赛。
- 看集锦即可的比赛。
- 可以跳过的比赛。
- 快速调整按钮，例如“轻松一点”“只看周末”“聚焦我的主队”。

### Match Detail 比赛详情页

目标：帮助用户判断是否观看某一场比赛，以及怎么看。

关键内容：

- 双方球队、日期、时间、比赛阶段。
- 重要性评分。
- 为什么这场比赛值得看。
- 关键球员或剧情点。
- 普通球迷也能看懂的解释。
- 提醒设置。
- 已结束比赛的赛后复盘区。

### Ask AI 问答页

目标：围绕观赛计划和比赛理解提供受控 AI 帮助。

示例问题：

- 今晚哪场最值得看？
- 我只关注阿根廷，接下来怎么安排？
- 我这周不能熬夜，帮我调整计划。
- 这场比赛为什么重要？
- 哪些比赛适合约朋友一起看？

### Settings 设置页

目标：允许用户修改偏好。

关键内容：

- 主队。
- 可看球时间。
- 提醒偏好。
- 浏览器通知权限状态。
- 当前数据源状态。

## Agent 工作流

MatchPilot 的 AI 能力不做成开放式闲聊，而是做成结构化 Agent 工作流。

### Profile Collector 用户画像收集

收集并保存用户偏好：

- 主队。
- 其他感兴趣的球队或球星。
- 可看球时间。
- 熬夜接受度。
- 看球深度。
- 社交观赛偏好。
- 提醒偏好。

### Match Ranker 比赛评分器

根据以下因素给比赛打分：

- 是否与主队相关。
- 比赛阶段。
- 球队实力或大众关注度。
- 开赛时间是否友好。
- 是否有宿敌、强强对话或话题性。
- 是否影响出线或晋级。
- 是否适合和朋友一起观看。

### Plan Generator 计划生成器

把评分后的比赛转化为观赛计划：

- 必看。
- 推荐观看。
- 看集锦即可。
- 可以跳过。

每个推荐都应该有简短理由，避免只给结论。

### Context Explainer 背景解释器

用普通球迷能理解的语言解释比赛：

- 这场为什么重要。
- 看球时应该关注什么。
- 关键球员或故事线。
- 对出线、晋级或后续赛程可能有什么影响。

### Reminder Agent 提醒 Agent

管理比赛提醒：

- 本地保存用户设置的提醒。
- 支持开赛前 10 分钟、30 分钟、1 小时提醒。
- 页面打开时触发站内提醒。
- 用户授权后可触发浏览器通知。
- 架构上预留未来服务端推送能力。

### Plan Adjuster 计划调整器

根据用户要求调整计划：

- 少看熬夜比赛。
- 更聚焦主队。
- 增加适合朋友局的比赛。
- 更适合新手球迷。
- 更关注高强度和高话题比赛。

## 数据策略

MVP 不依赖真实 API 也必须能稳定运行。

默认数据源：

- 本地 mock 世界杯比赛数据。
- 覆盖小组赛和淘汰赛示例，足够展示规划逻辑。

未来 API 设计：

- 抽象足球数据访问层。
- UI 和 Agent 逻辑不直接绑定某个数据供应商。
- 支持 mock 数据和外部 API 数据切换。
- 可考虑 football-data.org、API-Football 等数据源，具体取决于免费额度、覆盖范围和 API key 可用性。

适合接入的真实数据：

- 赛程。
- 球队。
- 比赛状态。
- 比分。
- 积分榜或出线形势。

MVP 避免接入：

- 分钟级实时文字直播。
- 赔率和博彩数据。
- 昂贵的高级球员指标。

## 提醒功能设计

MVP 提醒逻辑：

- 用户可以在比赛卡片或比赛详情页设置提醒。
- 提醒选项包括开赛前 10 分钟、30 分钟、1 小时。
- 提醒数据保存在 localStorage。
- 页面打开时定时检查即将触发的提醒。
- 到时间后显示 toast 或弹窗。
- 如果用户授权 Web Notification，也可以显示浏览器通知。

未来扩展：

- 后端数据库保存用户和提醒。
- 定时任务检查即将开始的比赛。
- 通过推送、邮件或小程序订阅消息提醒用户。

## 技术架构

推荐第一版技术栈：

- Vite React。
- TypeScript。
- Tailwind CSS。
- localStorage 保存用户偏好和提醒。
- mock 数据文件保存初始赛程。
- 可选 LLM 服务抽象。
- Vercel 部署，便于作品集展示。

核心模块：

- `data`：mock 赛程和未来 API adapter。
- `profile`：用户偏好存储。
- `scoring`：比赛评分规则。
- `planner`：观赛计划生成。
- `insights`：赛前解释和看点生成。
- `reminders`：本地提醒调度。
- `ai`：可选 LLM adapter 和 fallback 模板。
- `ui`：页面、卡片、底部导航和移动端布局。

## AI 接入策略

应用在没有 LLM API key 的情况下也应该可用。

Fallback 模式：

- 规则评分。
- 模板生成计划和看点。
- 输出稳定，适合 GitHub Demo。

LLM 模式：

- 把用户画像和比赛数据作为结构化输入。
- 生成更自然的计划解释。
- 回答 Ask AI 页面中的问题。
- 当没有实时数据时，避免编造事实。

这个设计可以保证项目稳定，同时展示 AI 产品和 Agent 架构思维。

## 成功标准

MVP 完成后，作品集评审者应该可以：

- 在桌面或手机浏览器打开应用。
- 选择主队和偏好。
- 看到个性化世界杯首页。
- 生成个人观赛计划。
- 打开比赛详情页。
- 设置比赛提醒并在提醒列表中看到。
- 使用 AI 式问答或快捷按钮调整计划。
- 通过文档理解产品策略和技术架构。

## 第一版实现默认选择

除非后续实现计划调整，第一版采用以下默认选择：

- 使用 Vite React 和 TypeScript。
- 第一版优先使用 mock 比赛数据，同时预留 API adapter。
- 先实现规则评分和模板式 AI fallback，核心 Demo 稳定后再考虑接入 LLM。
- 视觉风格参考现代体育 App：高对比、强比赛卡片、紧凑导航、清晰提醒控件。
